# Выполнил Толстиков С. ИВТ-13

# Практическое задание: Система управления личными задачами (Task Manager)

## Цель

Разработать на языке C приложение для управления личными задачами с фильтрами и сортировкой, демонстрирующее применение принципов ООП: инкапупляция, наследование и полиморфизм.

## Суть задания

Создать систему, которая:

- Позволяет добавлять задачи с названием, приоритетом и статусом
- Поддерживает два типа задач: простые задачи и задачи с дедлайном (датой завершения)
- Реализует полиморфные операции для всех типов задач
- Хранит список задач и предоставляет инструменты для их обработки

## Требования к системе

### Функциональные возможности

1. **Создание задач**
   - Добавление простых задач (без даты)
   - Добавление задач с дедлайном (с датой в формате dd-mm-yyyy)

2. **Управление статусами**
   - Три статусы: TODO (не выполнена), IN_PROGRESS (в процессе), DONE (выполнена)
   - Возможность изменять статус задачи

3. **Работа с приоритетами**
   - Три уровня: LOW (низкий), MEDIUM (средний), HIGH (высокий)
   - Сортировка задач по приоритету

4. **Фильтрация и поиск**
   - Вывод всех задач
   - Фильтр по статусу (например, только выполненные)
   - Поиск задачи с самым высоким приоритетом
   - Выявление задач с просроченным дедлайном (overdue)

### Принцип ООП для демонстрации

| Принцип | Как применить |
|---------|---------------|
| **Инкапсуляция** | Скрытые поля задач, доступ через методы |
| **Наследование** | Расширение базовой структуры задачи для задач с дедлайном |
| **Полиморфизм** | Единые операции `print()`, `isCompleted()` для разных типов задач |
